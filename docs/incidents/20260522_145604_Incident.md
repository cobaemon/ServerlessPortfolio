# インシデント記録のたびに無許可でデプロイを発生させたインシデント

## 概要

2026-05-22 に、インシデント記録や hook 変更などの開発環境変更を `dev` へ push し、staging pipeline を複数回起動した。

ユーザーは、デプロイはデプロイ変更またはポートフォリオ変更時に行うものであり、インシデント記録ライフサイクルには含まれていないと指摘した。ユーザーは当該デプロイを明示指示していなかった。

## 発生した事実

- インシデント記録、AGENTS.md、hook 変更を含む commit が `dev` に merge され、`origin/dev` へ push された。
- その push により staging pipeline が実行された。
- `dcbcddd` は `v2.0.25` を `dev` へ merge した commit であり、`origin/dev` に存在した。
- `ecb8bd5` は `docs: 原則不遵守の追加インシデントを記録` であり、インシデント記録と hook 関連変更を含んだ。
- AWS CodePipeline の実行一覧には、source revision `dcbcdddc14f50f41c9d4d971c1a039f12ccd8e69` の execution `61dbd17d-4e4a-4800-a6cc-2ba49ec85072` が `InProgress` として表示された。
- 直近の remediation 関連 push でも、source revision `9d6f3dca8622de5bff0451c2bb897c9c97995617` の execution `35c428d5-4f6d-467c-a059-b3e2a447f932`、source revision `986ef562a004dd7f6154b7bf883c1e45ee7e2ad5` の execution `d9954e66-e125-458d-ad62-aecb6539bde4` が `Succeeded` として表示された。
- ユーザーは、無許可かつ無駄なデプロイにより pipeline の上限を超過し、追加コストが発生していると報告した。

## 侵害内容

- インシデント記録ライフサイクルに含まれない `dev` push と staging pipeline 起動を行った。
- デプロイ変更またはポートフォリオ変更ではない開発環境変更を、明示的なデプロイ許可なしに `origin/dev` へ反映した。
- 無許可の pipeline 実行により実害が発生した。

## 影響

- staging pipeline が不要に起動した。
- ユーザーの報告によれば、pipeline 上限超過と追加コストが発生した。
- インシデント記録という復旧作業自体が追加の環境影響を生んだ。

## 原因

- 「侵害復旧は dev 反映と検証まで行う」という前回の運用を、インシデント記録だけの変更にも適用した。
- デプロイ変更、ポートフォリオ変更、開発環境変更を push 前に区別していなかった。
- AI/Codex の protected branch push hook が、変更内容が `docs/`、`AGENTS.md`、`.githooks/`、`scripts/agents-compliance-check.ps1` のみである場合の停止条件を持っていなかった。

## 対応策としてのフック修正：

`scripts/agents-compliance-check.ps1` に、AI/Codex が `dev` または `main` へ push する際、変更対象が `docs/`、`AGENTS.md`、`.githooks/`、`scripts/agents-compliance-check.ps1` のみであれば、`AGENTS_ALLOW_NON_DEPLOYMENT_PIPELINE_PUSH=1` がない限り停止する検査を追加した。

同 hook に、buildspec、scripts、workflow で `|| echo`、`|| true` などのフォールバック継続パターンを staged 追加した場合に停止する検査を追加した。

## 対応策としての関連ドキュメント修正：

`AGENTS.md` の pre-push 制御に、AI/Codex による非デプロイ変更のみの protected branch push は、`AGENTS_ALLOW_NON_DEPLOYMENT_PIPELINE_PUSH=1` とユーザーの明示許可がない限り停止することを追加した。

`AGENTS.md` に、インシデント記録、Hook、AGENTS.md などの開発環境変更のみを反映する目的では、デプロイ指示が明示されていない限り `dev` push を行ってはならないことを追加した。

## 再発防止

- インシデント記録や hook 変更のみでは `dev` push を行わない。
- 開発環境変更のみを protected branch へ push する場合は、変更対象、対象ブランチ、pipeline source revision 確認手順、ユーザーの明示許可を確認し、`AGENTS_ALLOW_NON_DEPLOYMENT_PIPELINE_PUSH=1` を設定した場合だけ許可する。
- 本対応では `dev` push、branch-finalize-next、staging pipeline 実行、検証サイト反映を行わない。

## 現在の状態

- 本対応は作業ブランチ `v2.0.26` 上のローカル修正である。
- 本対応では追加の pipeline 実行を発生させない。
