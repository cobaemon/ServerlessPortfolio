INC-20260526-003-重大

タイトル：再発防止策を特定インシデントに特化した内容で実装した

背景：ユーザーは、AGENTS や hook は原則を厳守させるものであり、特定のインシデントに特化した内容は不適切で、普遍的かつ汎用的でなければならないと指摘した。

発生インシデントの説明：

無断対応の中で、`docs/development-records/` 配下のデプロイ時間短縮記録に AWS 既存ログ確認結果を要求する、特定インシデントに寄った GitHook制御を追加した。

この対応は、デプロイ時間短縮という単一類型には作用するが、他の調査記録、方針策定、対応案作成で確認可能項目を確認しない違反には効かない。

既存制御にも、`docs/ai-progress/` 配下の staging pipeline / stack / site の検証完了主張だけを対象にする記述が残っていたため、外部環境の検証完了主張全般を対象にする汎用制御へ修復した。

定量記録：

- 影響を受けたブランチ数：1
- 作成した不適切な commit 数：0
- push 回数：0
- 起動または停止した pipeline 実行数：0
- 変更ファイル数：2
- 復旧 commit 数：0

なぜ発生したのか：

事実確認不足とスコープ変更が原因である。

元の指示範囲は、原則、指示、要件への違反を防ぐ普遍的な制御の実装だった。

不適切に狭めた範囲は、デプロイ時間短縮記録に AWS 既存ログ確認結果を要求する単一類型の制御だった。

確認すべきだった事実は、AGENTS の hook 設計が第一原則から実装制御までを原則のままの優先順位で対象にすること、再発防止策が同種または類似違反へ汎用的に効く必要があることだった。

再発防止として、AGENTS に「再発防止策は特定インシデント、特定ファイル、特定サービスだけに限定せず、同種または類似の違反を防ぐ汎用条件として設計する」ことを追加した。

何に違反したのか：

スコープ変更禁止の「ユーザーの明示指示を、一部の語句、単一の違反類型、または作業者に都合のよい範囲へ狭めてはならない」に違反した。

AGENTS の「原則厳守の hook 設計は、第一原則、第二原則、第三原則、第四原則、共通解釈規則、実行前制御、報告制御、実装制御を原則のままの優先順位で対象にする」に対し、単一類型へ狭めた。

発生時点：

再発防止策を設計し、制御系修正として実装する前の設計段階と、報告前段階で発生した。

本来作用すべき制御：

作業前制御で再発防止策の対象範囲を固定し、報告前制御で特定インシデント専用になっていないことを確認すべきだった。

実際に作用した制御：

単一インシデントに対する直接的な GitHook制御だけが作用し、汎用条件としての検証が作用しなかった。

制御が効かなかった理由：

再発防止策の有効範囲を同種・類似違反へ広げて検証しなかったため。

既存再発防止策が効かなかった理由：

既存策自体が特定インシデントへの反応として追加され、汎用性検査や作用タイミング検査を必須化していなかったため。

同種過去インシデント：

特定の staging pipeline やデプロイ時間短縮記録に寄った制御を追加した事象と同種である。

発生前に停止できたはずのゲート：

実行中制御で、再発防止策がデプロイ時間短縮記録という単一類型に限定されていることを検出していれば停止できた。

報告前制御で、対応案が同種または類似の違反へ効く汎用条件になっていないことを確認していれば停止できた。

再発防止策の作用タイミング：

制御設計前、制御系修正前、報告前に作用する。GitHook制御は実装された汎用条件の維持に作用する。

再発防止策の検証方法：

新しい制御が単一ファイル、単一サービス、単一インシデントだけでなく、同種または類似の違反に作用することを差分レビューで確認する。

制御区分：

非 Git 制御は作業前制御、実行中制御、報告前制御。GitHook制御は AGENTS 必須マーカーと staged diff 検査。

残存リスク：

汎用性の十分性は完全には機械判定できないため、報告前に作用範囲を明示して確認する必要が残る。

自己検知・自己報告の成否：

自己検知できず、ユーザー指摘後に発覚した。

制御不備または原則不遵守：

制御不備および原則不遵守である。

特定インシデント専用の検査を削除し、開発記録全般の確認対象・確認結果・未確認事項、外部環境の検証完了主張全般の確認文脈、AI/Codex commit の明示許可 guard に置き換えた。

対応策としての制御系修正：

`scripts/agents-compliance-check.ps1` から staging pipeline / docs/ai-progress に限定した検査を削除し、外部環境、pipeline、stack、site、デプロイ、検証サイトの検証完了または成功を記録する文書全般に、確認対象、確認結果、source revision / 対象差分 / commit / execution id のいずれかを要求する汎用検査を追加した。

`scripts/agents-compliance-check.ps1` に、開発記録全般へ確認対象、確認結果、未確認事項を要求する汎用検査を追加した。

対応策としての関連ドキュメント修正：

`AGENTS.md` に、再発防止策を特定インシデント、特定ファイル、特定サービスだけに限定せず、同種または類似の違反を防ぐ汎用条件として設計することを追加した。

`AGENTS.md`、`docs/ai-progress/README.md`、`docs/development-records/README.md` を、特定 staging 記録ではなく外部環境の検証完了主張全般と開発記録全般に効く記述へ修正した。
