INC-20260520-003-重大

タイトル：Phase 3 作業の責任範囲未達と完了扱い

背景：ユーザーは Phase 3 の権限最適化作業について、本番へ影響させず、検証で完結させ、安全第一で進めることを指示していた。権限最適化手順では、staging pipeline 検証を `dev` push による自動実行で行うこと、`branch-finalize-next` は明示指示時のみ実行することが記載されていた。

発生インシデントの説明：

`v2.0.7` 上で Phase 3 の `CloudFormationServiceRole` 権限縮小 commit を作成した後、`branch-finalize-next`、`dev` push、staging pipeline 検証が未実施の状態で、Phase 3 作業を実施したと報告した。

定量記録：

- 影響を受けたブランチ数：1（`v2.0.7`）
- 作成した不適切な commit 数：1（`5da3ab11ada5851504c598953359f26365fbd5f5`）
- push 回数：0
- 起動または停止した pipeline 実行数：0
- 変更ファイル数：1（`pipeline.yaml`）
- 復旧 commit 数：0

なぜ発生したのか：

手順書が、Phase 作業の完了条件として `branch-finalize-next`、`dev` push、pipeline source revision 確認、pipeline 実行状態確認までを明確に固定していなかった。加えて、ローカル commit とローカル検証をもって Phase 3 作業を完了扱いにした。

何に違反したのか：

- ユーザーが求めた、push による pipeline 検証動作確認までを責任範囲とする要求。
- 要件を1つでも満たしていない状態で、完了、成功、対応済み、問題なしと扱わないこと。
- 未確認事項は未確認と明記すること。

対応策としてのフック修正：

`scripts/agents-compliance-check.ps1` の pre-commit に、インシデント記録ファイル名、インシデント番号とレベル、`対応策としてのフック修正：`、`対応策としての関連ドキュメント修正：` の検査を追加する。

対応策としての関連ドキュメント修正：

`docs/incidents/README.md` に、インシデント対応は記録だけでは完了せず、対応策としての hook 修正と関連ドキュメント修正を責任範囲に含めることを追加する。`docs/iam-permission-optimization.md` に、各 Phase の作業責任範囲と完了条件として `branch-finalize-next`、`dev` push、pipeline source revision 確認、pipeline 実行状態確認を追加する。`AGENTS.md` に、インシデント記録形式の hook 制御と `dev` push 検証の完了条件を追加する。
