# インシデントサイクルの不完全実施

## 発生日時

2026-05-21 01:29:30

## 区分

違反

## 概要

`main` ブランチ上で実装した件のインシデント対応において、インシデント記録のみを作成し、hook と関連手順ドキュメントの修正を同時に行わなかった。

さらに、インシデント記録の再発防止には具体策を記載したにもかかわらず、`対応策としての制御系修正：` と `対応策としての関連ドキュメント修正：` に `未実施` と記載した。

## 影響範囲

- `docs/incidents/20260521_012509_Incident.md`
- `scripts/agents-compliance-check.ps1`
- `AGENTS.md`

## 確認した事実

- `docs/incidents/20260521_012509_Incident.md` に `対応策としての制御系修正：` が存在した。
- `docs/incidents/20260521_012509_Incident.md` の `対応策としての制御系修正：` に `未実施` と記載されていた。
- `docs/incidents/20260521_012509_Incident.md` に `対応策としての関連ドキュメント修正：` が存在した。
- `docs/incidents/20260521_012509_Incident.md` の `対応策としての関連ドキュメント修正：` に `未実施` と記載されていた。
- `scripts/agents-compliance-check.ps1` は、インシデント記録に必須見出しが存在するかだけを検査していた。
- `AGENTS.md` は、必須見出しの存在を停止条件としていたが、見出し本文が `未実施` の場合に停止する条件を明記していなかった。

## 原因

インシデントサイクルを、インシデント記録の作成だけで完了扱いした。  
また、hook と手順ドキュメントの不足を再発防止策に反映する責任範囲を実行しなかった。

## 即時対応

- `docs/incidents/20260521_012509_Incident.md` の `未実施` 記載を、実際の 制御系修正内容と関連ドキュメント修正内容へ置き換える。
- `scripts/agents-compliance-check.ps1` に、インシデント記録の corrective action section が空または `未実施` の場合に停止する検査を追加する。
- `scripts/agents-compliance-check.ps1` に、インシデント記録を staged に含める場合は `scripts/agents-compliance-check.ps1` と `AGENTS.md` も同じ staged に含める検査を追加する。
- `AGENTS.md` の `pre-commit` 手順に上記停止条件を追記する。

## 再発防止

- インシデント記録を作成する場合、記録だけで完了扱いしない。
- 制御系修正と関連手順ドキュメント修正を同じ変更単位に含める。
- corrective action section に `未実施` と記載したインシデント記録を GitHook制御で停止する。
- インシデント記録、hook、手順ドキュメントを同時に確認してから報告する。

## 対応策としての制御系修正：

`scripts/agents-compliance-check.ps1` の `Assert-IncidentRecordCycleDocuments` を強化し、インシデント記録の corrective action section が空または `未実施` の場合に停止する検査を追加する。

また、インシデント記録が staged に含まれる場合、`scripts/agents-compliance-check.ps1` と `AGENTS.md` が同じ staged に含まれない場合に停止する検査を追加する。

## 対応策としての関連ドキュメント修正：

`AGENTS.md` の `pre-commit` 手順に、インシデント記録の corrective action section が空または `未実施` の場合に停止することを追記する。

また、インシデント記録を staged に含める場合、hook 本体と手順ドキュメントの修正を同じ staged に含める必要があることを追記する。

## 未確認事項

- 修正後 GitHook制御を staged 状態で実行した結果。
- 最終 commit 対象に含める未追跡ディレクトリの範囲。

発生時点：

2026-05-28 の復旧作業時点で、既存記録本文に基づき追補した。発生時点は、既存記録の「発生インシデントの説明：」および「なぜ発生したのか：」に記載された手順逸脱、完了誤認、確認漏れ、または無許可操作の発生時点である。

本来作用すべき制御：

応答前制御、作業前制御、ツール実行前制御、実行中制御、報告前制御、事後監査制御が作用すべきだった。Git 状態、staged diff、commit message で決定的に判定できる事項は GitHook制御が作用すべきだった。

実際に作用した制御：

既存記録本文では、発生前に停止した制御は記録されていない。発生後に、インシデント記録、制御系修正、関連ドキュメント修正が事後対応として記録されている。

制御が効かなかった理由：

既存記録本文に記載された完了条件、確認対象、作業対象、または記録条件が、発生前の応答、作業、ツール実行、報告、commit の各時点で停止条件として固定されていなかったため。

既存再発防止策が効かなかった理由：

既存再発防止策は、同種の原則違反を発生前に止めるための作用タイミング、確認対象、検証方法を各記録内で十分に固定していなかったため。

同種過去インシデント：

2026-05-20 から 2026-05-28 までに、確認漏れ、スコープ逸脱、完了誤認、制御不備、記録不備に関するインシデント記録が `docs/incidents/` に複数存在する。

発生前に停止できたはずのゲート：

応答前制御の指示対象固定ゲート、作業前制御の確認対象列挙ゲート、ツール実行前制御の対象範囲確認ゲート、実行中制御のスコープ逸脱停止ゲート、報告前制御の完了条件確認ゲート、事後監査制御の自己検知ゲート。

再発防止策の作用タイミング：

応答前、作業前、ツール実行前、実行中、報告前、commit 前、commit message 作成時、push 前、インシデント記録作成時。

再発防止策の検証方法：

該当記録が、発生時点、本来作用すべき制御、実際に作用した制御、制御が効かなかった理由、既存再発防止策が効かなかった理由、同種過去インシデント、発生前に停止できたはずのゲート、作用タイミング、検証方法、制御区分、残存リスク、自己検知・自己報告の成否を持つことを pre-commit Hook で確認する。

制御区分：

応答前制御、作業前制御、ツール実行前制御、実行中制御、報告前制御、GitHook制御、事後監査制御。

残存リスク：

チャット上の指示解釈、質問と作業指示の識別、外部環境確認、実行結果確認は、Hook だけでは完全に停止できない。非 Git 制御で作業前および報告前に確認する必要が残る。

自己検知・自己報告の成否：

既存記録本文では、発生前または発生直後に AI/Codex が自己検知して自己報告した事実は記録されていない。事後にインシデント記録として整理されている。
