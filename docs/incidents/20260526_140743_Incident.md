INC-20260526-006-重大

タイトル：指示を曲解してインシデント記録を削除対象に含めようとした

背景：ユーザーは「私はレビュー文書のファイル化を許可していないし指示もしていない インシデント記録とファイルを削除しチャットで表示しろ」と発言した。その後、ユーザーは意図を「インシデント記録を行い、記録後に作成したレビューファイルを削除して内容をチャットで表示しろ」と明確化した。

発生インシデントの説明：

ユーザーの意図は、不正に作成したレビュー文書についてインシデント記録を行い、その後にレビュー文書だけを削除し、レビュー文書の内容をチャットに表示することだった。

それにもかかわらず、発言内の「インシデント記録とファイルを削除し」を、インシデント記録ファイル自体も削除対象に含める意味として扱い、削除対象確認を開始した。

定量記録：

- 影響を受けたブランチ数：1
- 作成した不適切な commit 数：0
- push 回数：0
- 起動または停止した pipeline 実行数：0
- 変更ファイル数：0
- 復旧 commit 数：0

なぜ発生したのか：

事実確認不足が原因である。

確認すべきだった事実は、削除対象が「作成したレビューファイル」のみなのか、「インシデント記録ファイル」を含むのかだった。

曖昧な文を確認せず、削除対象を広げる解釈を採用した。

再発防止として、削除、復旧、ロールバック、移動、上書きなど破壊的または不可逆に近い操作では、対象ファイルの一覧を確定し、曖昧な場合は作業前に確認する。特にインシデント記録、AGENTS、Hook、ドキュメント規則に関係するファイルは、明示された対象だけを扱う。

何に違反したのか：

第一原則の「不明点や曖昧な点があれば、作業前に必ず確認すること」「明示的に許可がない限り、作業を行わないこと」に違反した。

共通解釈規則の「指示文中の語句を、確認なしに一般化、短縮、言い換え、補完しないこと」「指示対象を、確認なしに別対象へ拡張しないこと」「確認が必要な状態で、確認なしに作業を続行しないこと」に違反した。

スコープ変更禁止の「ユーザーの明示指示を、一部の語句、単一の違反類型、または作業者に都合のよい範囲へ狭めてはならない」に対し、削除対象を確認せず別対象へ拡張した点で違反した。

発生時点：

削除対象を確認する前の応答前段階と、削除対象確認を開始した作業前段階で発生した。

本来作用すべき制御：

応答前制御で削除対象が複数候補になることを検出し、作業前制御で対象ファイル一覧を確定すべきだった。

実際に作用した制御：

削除対象を確定する制御が作用せず、インシデント記録ファイルまで削除候補に含める解釈を採用した。

制御が効かなかった理由：

曖昧な削除指示を破壊的操作として扱わず、確認前に対象を広げたため。

既存再発防止策が効かなかった理由：

既存策は削除操作の staging 後や commit 前に偏っており、削除対象解釈の時点で作用しなかったため。

同種過去インシデント：

指示対象を確認なしに拡張したインシデント、質問を作業指示として扱ったインシデントと同種である。

発生前に停止できたはずのゲート：

応答前制御で、ユーザー発言の削除対象が複数候補になることを検出していれば停止できた。

作業前制御で、削除対象が `docs/development-records/agents-hook-incident-review.md` のみなのか、インシデント記録ファイルを含むのかを確定していれば停止できた。

再発防止策の作用タイミング：

応答前、作業前、削除前、報告前に作用する。

再発防止策の検証方法：

削除、ロールバック、移動、上書きでは、対象ファイル一覧と操作内容が明示されていることを確認する。

制御区分：

非 Git 制御は応答前制御、作業前制御、ファイル編集前制御。GitHook制御は staged deletion の補助確認に限定される。

残存リスク：

削除対象の会話解釈は応答前制御、作業前制御、ファイル編集前制御で停止する必要がある。

自己検知・自己報告の成否：

自己検知できず、ユーザー指摘後に発覚した。

制御不備または原則不遵守：

原則不遵守である。

削除対象を確定する前の会話解釈は、応答前制御、作業前制御、ファイル編集前制御で停止すべきである。

ただし、Git 管理済みファイルの削除が staged された後であれば、staged diff から削除内容を確認できる。今回の時点では削除は実行されておらず、staged もされていない。

対応策としての制御系修正：

作業開始前の削除対象解釈は応答前制御と作業前制御で停止し、削除直前はファイル編集前制御で対象パスを確定する。

対応策としての関連ドキュメント修正：

`AGENTS.md` に制御系0設計、応答前制御、作業前制御、ツール実行前制御、ファイル編集前制御、実行中制御、報告前制御、事後監査制御を追加し、曖昧な削除対象を作業前制御とファイル編集前制御で停止する制御として明文化する。

元の指示範囲：

不正に作成したレビュー文書についてインシデント記録を行い、記録後に作成したレビューファイルを削除し、レビュー文書の内容をチャットに表示すること。

不適切に狭めた範囲：

削除対象を「作成したレビューファイル」に固定せず、インシデント記録ファイルまで削除候補に含める解釈へ変更した範囲。

再発防止：

削除、復旧、ロールバック、移動、上書きでは、作業前に対象ファイル一覧を固定し、複数候補がある場合は確認する。インシデント記録は、削除対象として明示されていない限り削除対象に含めない。
