# 正規手順に含まれる branch-finalize-next 実行前に作業中断したインシデント

## 概要

2026-05-21 に P2 IAM 権限最適化作業について、ユーザーは「P2作業の状況確認、適切なロールバック、正規手順での作業の再開、検証サイトでの検証」を指示した。

この指示は、既存手順で staging pipeline 検証の責任範囲に含まれる `branch-finalize-next`、`dev` 反映、pipeline source revision 確認、検証サイト確認まで継続する作業だった。それにもかかわらず、作業ブランチ `v2.0.18` で commit を作成した時点で停止し、`branch-finalize-next` 実行をユーザーへ再要求した。

## 発生した事実

- P2 作業 commit `991f013473990feb13aee9ddafa56e969fd4b3fa` を `v2.0.18` 上で作成した。
- その後、`branch-finalize-next` を実行せずに停止した。
- `docs/ai-progress/iam-permission-optimization.md` は、staging pipeline 検証を含む Phase で `branch-finalize-next`、`dev` push、pipeline source revision 確認、pipeline 実行状態確認を責任範囲に含めていた。
- `AGENTS.md` は `branch-finalize-next` の実行条件を狭く記述していたが、ユーザーの「正規手順での作業の再開」「検証サイトでの検証」という指示を責任範囲に含む明示指示として扱わなかった。

## 侵害内容

- 正規手順として指定済みの `branch-finalize-next` を実行せず、不正な作業中断を発生させた。
- ユーザーに追加指示を要求し、不要な時間とコストを発生させた。
- P2 の検証サイト確認を遅延させた。

## 影響

- P2 の正規検証が予定より遅延した。
- ユーザーが再指摘しなければ、作業ブランチ commit のみで停止した状態が継続していた。
- 手順書間の解釈差が、実作業の中断として顕在化した。

## 原因

- `AGENTS.md` の「ユーザーが明示的に `branch-finalize-next` の実行を指示した場合のみ実行する」を、正規手順での作業再開や検証サイトでの検証依頼より優先して狭く解釈した。
- `docs/ai-progress/iam-permission-optimization.md` の責任範囲に含まれる `branch-finalize-next` を、実行済みにすべき作業ではなく追加確認事項として扱った。
- 作業責任範囲に含まれる手順を最後まで実行するという確認を、final response 前に行わなかった。

## 対応策としての制御系修正：

`scripts/agents-compliance-check.ps1` の AGENTS.md 必須マーカーに「検証サイトでの検証または正規手順での作業再開を依頼された場合はbranch-finalize-nextを責任範囲に含めること」を追加した。

このマーカーにより、当該責任範囲の規則が AGENTS.md から削除された状態では GitHook制御が通らない。

## 対応策としての関連ドキュメント修正：

`AGENTS.md` に、検証サイトでの検証、正規手順での作業再開、staging pipeline 検証、または `dev` 反映後の検証を依頼された場合、その依頼は `branch-finalize-next` 実行を責任範囲に含む明示指示として扱い、ローカル commit 後に `branch-finalize-next` 実行前で停止してはならないことを追記した。

`docs/ai-progress/iam-permission-optimization.md` の各 Phase 手順から、`branch-finalize-next` を追加確認事項として扱う記述を改め、検証サイトでの検証、正規手順での作業再開、staging pipeline 検証、または `dev` 反映後の検証依頼がある場合は `branch-finalize-next` を実行することを追記した。
