# インシデント記録

このディレクトリは、作業中に発生した指示違反、手順逸脱、影響発生、復旧対応を記録します。

## インシデント対応サイクル

インシデント対応は、記録ファイルの作成だけでは完了しません。

該当インシデントの対応策として、次を責任範囲に含めます。

- インシデント記録ファイルを作成する。
- 違反した指示、手順、停止条件、完了条件を特定する。
- hook で機械的に停止できる違反は、`.githooks` または `scripts/agents-compliance-check.ps1` を修正する。
- 手順書、運用ドキュメント、関連ドキュメントに不足または曖昧さがある場合は修正する。
- hook 修正または関連ドキュメント修正が不要、または実装不能な場合は、その理由をインシデント記録に明記する。

インシデント対応を完了扱いにできるのは、インシデント記録、hook 修正要否、関連ドキュメント修正要否の確認がすべて記録されている場合に限ります。

## ファイル名

インシデント記録ファイルは次の形式で作成します。

```text
{作成時のタイムスタンプ}_Incident.md
```

タイムスタンプは Windows ファイル名で使用できる `yyyyMMdd_HHmmss` 形式とします。

## インシデント番号

インシデント番号は次の形式で採番します。

```text
INC-{yyyyMMdd}-{3桁連番}
```

同一日に複数発生した場合は、作成順に `001` から 1 ずつ増やします。

## インシデントレベル

インシデントレベルは次の降順で分類します。

| レベル | 分類基準 |
| --- | --- |
| 致命的 | 復元不可 |
| 侵害 | 実際に実害が発生 |
| 重大 | 複数の違反が同時多発 |
| 違反 | 単発の違反 |

## 記録形式

各インシデントファイルは次の形式で記録します。

```text
インシデント番号-インシデントレベル

タイトル：{インシデント概要}

背景：{指示や要件}

発生インシデントの説明：

なぜ発生したのか：

何に違反したのか：

対応策としてのフック修正：

対応策としての関連ドキュメント修正：
```

## 定量記録項目

発生インシデントの説明には、確認できる範囲で次の数値を含めます。

- 影響を受けたブランチ数
- 作成した不適切な commit 数
- push 回数
- 起動または停止した pipeline 実行数
- 変更ファイル数
- 復旧 commit 数

## 定性記録項目

発生インシデントの説明、なぜ発生したのか、何に違反したのかには、確認できる範囲で次を含めます。

- 逸脱した指示または手順
- 発生した影響
- 復旧方法
- 残存リスク
- 再発防止のために必要な制御
- 対応策として実施した hook 修正、または修正不要と判断した理由
- 対応策として実施した関連ドキュメント修正、または修正不要と判断した理由
