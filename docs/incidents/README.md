# インシデント記録

このディレクトリは、原則違反、指示違反、手順逸脱、影響発生、復旧対応を記録する。

## インシデント定義

- 原則への直接違反または間接違反
- ユーザー指示への違反
- 正規手順または制御手順からの逸脱
- 実環境、作業ツリー、報告、判断、時間、コスト、トークンへの影響
- 復旧対応、再発防止策、制御不備の記録対象

## レベル

| レベル | 分類基準 |
| --- | --- |
| 致命的 | 復元不可、または長期かつ根本的な制御不全 |
| 侵害 | 実害が発生 |
| 重大 | 複数の違反が同時多発 |
| 違反 | 単発の違反 |

## 必須項目

インシデント記録には次を含める。

- 発生時点：
- 本来作用すべき制御：
- 実際に作用した制御：
- 制御が効かなかった理由：
- 既存再発防止策が効かなかった理由：
- 発生前に停止できたはずのゲート：
- 制御区分：
- 対応策としての制御系修正：
- 対応策としての関連ドキュメント修正：

## 制御

GitHook は、staged されたインシデント記録に必須項目がない場合に停止する。
CodexHook は、インシデント対応を完了報告する前に、未検証、未記録、未実装を完了扱いしないよう停止する。
