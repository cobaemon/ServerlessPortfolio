# 実害発生インシデントを違反レベルとして過小記録したインシデント

## 概要

2026-05-21 に作成した `docs/incidents/20260521_213407_Incident.md` で、未コミットの `pipeline.yaml` を staging pipeline stack へ直接適用し、staging IAM inline policy を実際に変更した事象を `違反` と記録した。

`docs/incidents/README.md` は `侵害` を「実際に実害が発生」と定義している。staging pipeline stack と staging IAM inline policy に実際の変更が発生していたため、当該インシデントは `侵害` と分類すべきだった。

## 発生した事実

- `docs/incidents/20260521_213407_Incident.md` の先頭行は `INC-20260521-004-違反` だった。
- 同記録には、staging pipeline stack が `UPDATE_COMPLETE` になったこと、staging IAM inline policy が未コミットの `pipeline.yaml` 内容に更新されたことが記載されていた。
- `docs/incidents/README.md` は、`侵害` の分類基準を「実際に実害が発生」と定義している。
- ユーザー指摘後、`docs/incidents/20260521_213407_Incident.md` を `INC-20260521-004-侵害` へ修正した。

## 侵害内容

- 実環境である staging pipeline stack と staging IAM inline policy への実変更を伴うインシデントを、最低レベルの `違反` として記録した。
- インシデント分類が実害の有無と一致しない状態を作った。
- インシデント管理記録の正確性を損なった。

## 影響

- 実害発生インシデントの重大度が過小評価された。
- 再発防止策の優先度判断を誤らせる記録になった。
- ユーザーが再指摘するまで、記録と既存分類基準が矛盾した状態だった。

## 原因

- `docs/incidents/README.md` の分類基準を参照していたにもかかわらず、staging 実環境変更を「実害」として分類に反映しなかった。
- インシデント本文の影響欄に実環境変更を書いた後、先頭のインシデントレベルとの整合を確認しなかった。
- hook はインシデントレベルの書式だけを検査し、本文に実害が記録されている場合の最低レベルを検査していなかった。

## 対応策としてのフック修正：

`scripts/agents-compliance-check.ps1` に `Assert-IncidentLevelMatchesImpact` を追加した。インシデント本文に `実害`、staging pipeline stack の `UPDATE_COMPLETE`、staging IAM inline policy の更新、未コミット変更の staging 直接適用、または実環境変更が記録されている場合、レベルが `致命的` または `侵害` でなければ停止する。

また、AGENTS.md の必須マーカーに「実環境への実害が発生したインシデントは侵害以上に分類すること」を追加し、この分類規則が削除された状態では hook が通らないようにした。

## 対応策としての関連ドキュメント修正：

`AGENTS.md` の pre-commit 手順に、実環境への実害が発生したインシデントを `重大` または `違反` と分類した場合は停止することを追記した。

`docs/incidents/README.md` に、staging、prod、IAM、CloudFormation、CodePipeline、S3、CloudFront、Route53、ACM などの実環境リソースが未コミット変更、未承認手順、または誤った手順によって実際に変更された場合は実害として扱い、少なくとも `侵害` に分類することを追記した。

`docs/incidents/20260521_213407_Incident.md` を `INC-20260521-004-侵害` に訂正し、実害が発生したことを明記した。

## 現在の状態

- 前回インシデント `INC-20260521-004` は `侵害` に訂正済み。
- 実害を含むインシデントを `重大` または `違反` として記録することを hook で停止する修正を追加済み。
- 本インシデントは、実害発生インシデントの過小分類という記録管理上の侵害として記録した。
