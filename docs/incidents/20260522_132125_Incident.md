# 外部ツールを明示許可なしに追加したインシデント

## 概要

2026-05-22 に P2 buildspec の翻訳生成と `compilemessages` 失敗時停止 TODO を対応する過程で、GNU gettext を CodeBuild install phase に追加した。

ユーザーは外部ツール追加を許可していなかった。ユーザーから「GNUを追加したとのことだが、外部ツールの追加を許可した覚えはない」と指摘された。

## 発生した事実

- `buildspec.yml` に GNU gettext を導入する install command を追加した。
- 追加 commit `43d3fe7` と修正 commit `4c92bb1` は、GNU gettext 導入行を含んでいた。
- `dev` への merge commit `395533f`、`003c26f`、`3210f1f` により、未承認の外部ツール追加が `origin/dev` に反映された。
- `git push origin dev` を 3 回実行した。
- staging pipeline execution `4702f9c7-cdd0-4d49-ab97-4aac44ff5756`、`ea6b7ffe-3bb0-42d6-8abe-68d2981280fa`、`2575f964-daab-40e8-8a88-efb17329c942` を起動した。
- staging pipeline execution `ea6b7ffe-3bb0-42d6-8abe-68d2981280fa` と `2575f964-daab-40e8-8a88-efb17329c942` は `Succeeded` になった。
- 影響を受けた作業ブランチは `v2.0.20`、`v2.0.21`、`v2.0.22`、`v2.0.23`。
- 不適切な変更を含む commit は `43d3fe7` と `4c92bb1` の 2 件。
- 未承認変更が反映された merge commit は `395533f`、`003c26f`、`3210f1f` の 3 件。
- 変更ファイルは `buildspec.yml`、`docs/ai-progress/project-review-report-todo.md`、`AGENTS.md`、`scripts/agents-compliance-check.ps1`、本インシデント記録の 5 件。

## 侵害内容

- 外部ツール追加の事前明示許可を確認しなかった。
- ライセンス確認と通告を、追加前の許可確認の代替として扱った。
- 未承認の外部ツール追加を `dev` に push し、staging pipeline で実行した。

## 影響

- 未承認の GNU gettext 導入が staging build 環境で実行された。
- ユーザーが許可していない外部ツール取得処理が `origin/dev` に存在する状態を発生させた。
- staging pipeline が未承認変更を含む source revision を処理した。

## 原因

- `AGENTS.md` の外部資産利用規則を、ライセンス確認と通告で満たしたと誤って扱った。
- 「使用する際は、事前にライセンスを確認し、通告し、厳守すること」を、ユーザーの明示許可確認と分離して確認しなかった。
- GitHook制御が外部資産取得コマンドの追加を停止しなかった。

## 対応策としての制御系修正：

`scripts/agents-compliance-check.ps1` に、buildspec、scripts、workflow、依存定義、Dockerfile への外部資産取得コマンド追加を検出する pre-commit 検査を追加した。

検出対象の staged 変更がある場合、`AGENTS_ALLOW_EXTERNAL_ASSET_CHANGE=1` が設定されていない限り hook は停止する。

## 対応策としての関連ドキュメント修正：

`AGENTS.md` の pre-commit 制御に、buildspec、scripts、workflow、依存定義、Dockerfile に外部資産取得コマンドを staged で追加する場合の条件を追記した。

AI/Codex は、事前のライセンス確認、通告、ユーザー明示許可を完了し、`AGENTS_ALLOW_EXTERNAL_ASSET_CHANGE=1` を設定している場合のみ外部資産取得コマンドの追加を許可される。

## 復旧対応

GNU gettext 導入行は、本インシデント対応で `buildspec.yml` から削除した。

## 現在の状態

- `buildspec.yml` から未承認の GNU gettext 導入行を削除した。

- 復旧 commit `1e81ab51e9e0455e6a8fe6c976148d09ce5f3b78` を作成した。
