INC-20260522-005-重大

# 原則厳守 hook のスコープを不当に狭めたインシデント

## 概要

2026-05-22 に、ユーザーは「すべての原則を厳守させる hook」を指示していた。

しかし対応では、原則全体ではなく「未確認・推測」など一部の違反類型を中心に hook の検査対象を追加した。これは、ユーザー指示のスコープを勝手に狭める行為であり、共通解釈規則および第四原則に違反していた。

## 発生した事実

- ユーザーは「すべての原則を厳守させるHookを設計実装しろ」と指示した。
- 既存の `scripts/agents-compliance-check.ps1` は、AGENTS.md 必須マーカー、成果物ドキュメントの evidence label、インシデント記録形式、staging source revision、外部資産取得、推測を含むインシデント記録、hook 不備を含むインシデント記録を検査していた。
- 既存の `scripts/agents-compliance-check.ps1` は、AGENTS.md の第一原則から第四原則、共通解釈規則、実行前制御、報告制御、実装制御、スコープ変更禁止を一体として確認する commit message gate を持っていなかった。
- 既存の `AGENTS.md` は、ユーザーが提示した実行前制御、報告制御、実装制御、スコープ変更禁止の詳細項目を十分に含んでいなかった。
- その結果、hook 不備または原則不遵守の再発防止が、一部の違反類型に偏った。

## 侵害内容

- 元の指示範囲は、第一原則、第二原則、第三原則、第四原則、共通解釈規則、実行前制御、報告制御、実装制御を原則のままの優先順位で厳守させる hook 設計だった。
- 不適切に狭めた範囲は、未確認、推測、外部資産取得、incident hook 不備など一部の検査項目だった。
- 指示を確認なしに一部の語句へ限定した。
- 原則厳守を目的とした hook が、原則全体を対象にする設計になっていなかった。

## 影響

- 原則違反の再発防止が、ユーザーの指示範囲より狭い範囲に限定された。
- 勝手なスコープ変更という禁止事項そのものを hook が検査していなかった。
- ユーザーが再指摘するまで、原則全体を対象にした確認 gate の不足が残った。

## 原因

- 事実確認不足。
- 確認すべきだった事実は、ユーザーの指示範囲が「未確認・推測」ではなく「すべての原則を厳守させる Hook」であること、既存 hook が原則全体の commit message gate を持っていないこと、AGENTS.md に実行前制御、報告制御、実装制御、スコープ変更禁止が不足していることである。
- 指示文の「すべての原則」を、確認なしに一部の直近違反類型へ置き換えた。
- 機械的に停止できる項目と、commit message で確認させる項目を分離して設計していなかった。
- 原則全体を優先順位どおりに確認するための commit message marker を hook に実装していなかった。

## 対応策としてのフック修正：

`scripts/agents-compliance-check.ps1` に、AGENTS.md の第一原則、第二原則、第三原則、第四原則、共通解釈規則、実行前制御、報告制御、実装制御、スコープ変更禁止の必須マーカー確認を追加した。

同 hook に、branch-finalize-next の protected branch merge commit を除く commit message に `原則確認:`、`第一原則:`、`第二原則:`、`第三原則:`、`第四原則:`、`共通解釈規則:`、`実行前制御:`、`報告制御:`、`実装制御:`、`スコープ変更なし:`、`外部資産:` を要求する検査を追加した。

同 hook に、スコープ変更または指示範囲の不当な限定を記録するインシデントで、元の指示範囲、不適切に狭めた範囲、再発防止策を要求する検査を追加した。

## 対応策としての関連ドキュメント修正：

`AGENTS.md` に、実行前制御、報告制御、実装制御、スコープ変更禁止を追加した。

`AGENTS.md` の Hook制御に、原則全体の必須マーカー確認、commit message の原則確認 gate、スコープ変更インシデントの記録要件を追加した。

## 再発防止

- 原則全体を AGENTS.md の必須マーカーとして検査する。
- commit message に原則全体の確認項目を要求し、単一違反類型への偏りを commit 時点で停止する。
- スコープ変更インシデントには、元の指示範囲と不適切に狭めた範囲を明記させる。

## 現在の状態

- 本対応は作業ブランチ `v2.0.26` 上のローカル修正である。
- 本対応では `dev` push、branch-finalize-next、staging pipeline 実行、検証サイト反映を行わない。
