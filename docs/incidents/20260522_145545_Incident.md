# 原則厳守 hook のスコープを不当に狭めたインシデント

## 概要

2026-05-22 に、ユーザーは「すべての原則を厳守させる hook」を指示していた。

しかし対応では、原則全体ではなく「未確認・推測」など一部の違反類型を中心に hook の検査対象を追加した。これは、ユーザー指示のスコープを勝手に狭める行為であり、共通解釈規則および第四原則に違反していた。

## 発生した事実

- ユーザーは「すべての原則を厳守させるHookを設計実装しろ」と指示した。
- 既存の `scripts/agents-compliance-check.ps1` は、AGENTS.md 必須マーカー、成果物ドキュメントの evidence label、インシデント記録形式、staging source revision、外部資産取得、推測を含むインシデント記録、制御不備を含むインシデント記録を検査していた。
- 既存の `scripts/agents-compliance-check.ps1` は、AGENTS.md の第一原則から第四原則、共通解釈規則、実行前制御、報告制御、実装制御、スコープ変更禁止を一体として確認する commit message gate を持っていなかった。
- 既存の `AGENTS.md` は、ユーザーが提示した実行前制御、報告制御、実装制御、スコープ変更禁止の詳細項目を十分に含んでいなかった。
- その結果、制御不備または原則不遵守の再発防止が、一部の違反類型に偏った。

## 侵害内容

- 元の指示範囲は、第一原則、第二原則、第三原則、第四原則、共通解釈規則、実行前制御、報告制御、実装制御を原則のままの優先順位で厳守させる hook 設計だった。
- 不適切に狭めた範囲は、未確認、推測、外部資産取得、incident 制御不備など一部の検査項目だった。
- 指示を確認なしに一部の語句へ限定した。
- 原則厳守を目的とした GitHook制御が、原則全体を対象にする設計になっていなかった。

## 影響

- 原則違反の再発防止が、ユーザーの指示範囲より狭い範囲に限定された。
- 勝手なスコープ変更という禁止事項そのものを GitHook制御が検査していなかった。
- ユーザーが再指摘するまで、原則全体を対象にした確認 gate の不足が残った。

## 原因

- 事実確認不足。
- 確認すべきだった事実は、ユーザーの指示範囲が「未確認・推測」ではなく「すべての原則を厳守させる制御系」であること、既存GitHook制御 が原則全体の commit message gate を持っていないこと、AGENTS.md に実行前制御、報告制御、実装制御、スコープ変更禁止が不足していることである。
- 指示文の「すべての原則」を、確認なしに一部の直近違反類型へ置き換えた。
- 該当制御レイヤで停止できる項目と、commit message で確認させる項目を分離して設計していなかった。
- 原則全体を優先順位どおりに確認するための commit message marker を GitHook制御に実装していなかった。

## 対応策としての制御系修正：

`scripts/agents-compliance-check.ps1` に、AGENTS.md の第一原則、第二原則、第三原則、第四原則、共通解釈規則、実行前制御、報告制御、実装制御、スコープ変更禁止の必須マーカー確認を追加した。

同 GitHook制御に、branch-finalize-next の protected branch merge commit を除く commit message に `原則確認:`、`第一原則:`、`第二原則:`、`第三原則:`、`第四原則:`、`共通解釈規則:`、`実行前制御:`、`報告制御:`、`実装制御:`、`スコープ変更なし:`、`外部資産:` を要求する検査を追加した。

同 GitHook制御に、スコープ変更または指示範囲の不当な限定を記録するインシデントで、元の指示範囲、不適切に狭めた範囲、再発防止策を要求する検査を追加した。

## 対応策としての関連ドキュメント修正：

`AGENTS.md` に、実行前制御、報告制御、実装制御、スコープ変更禁止を追加した。

`AGENTS.md` の Hook制御に、原則全体の必須マーカー確認、commit message の原則確認 gate、スコープ変更インシデントの記録要件を追加した。

## 再発防止

- 原則全体を AGENTS.md の必須マーカーとして検査する。
- commit message に原則全体の確認項目を要求し、単一違反類型への偏りを commit 時点で停止する。
- スコープ変更インシデントには、元の指示範囲と不適切に狭めた範囲を明記させる。

## 現在の状態

- 本対応は作業ブランチ `v2.0.26` 上のローカル修正である。
- 本対応では `dev` push、branch-finalize-next、staging pipeline 実行、検証サイト反映を行わない。

発生時点：

2026-05-28 の復旧作業時点で、既存記録本文に基づき追補した。発生時点は、既存記録の「発生インシデントの説明：」および「なぜ発生したのか：」に記載された手順逸脱、完了誤認、確認漏れ、または無許可操作の発生時点である。

本来作用すべき制御：

応答前制御、作業前制御、ツール実行前制御、実行中制御、報告前制御、事後監査制御が作用すべきだった。Git 状態、staged diff、commit message で決定的に判定できる事項は GitHook制御が作用すべきだった。

実際に作用した制御：

既存記録本文では、発生前に停止した制御は記録されていない。発生後に、インシデント記録、制御系修正、関連ドキュメント修正が事後対応として記録されている。

制御が効かなかった理由：

既存記録本文に記載された完了条件、確認対象、作業対象、または記録条件が、発生前の応答、作業、ツール実行、報告、commit の各時点で停止条件として固定されていなかったため。

既存再発防止策が効かなかった理由：

既存再発防止策は、同種の原則違反を発生前に止めるための作用タイミング、確認対象、検証方法を各記録内で十分に固定していなかったため。

同種過去インシデント：

2026-05-20 から 2026-05-28 までに、確認漏れ、スコープ逸脱、完了誤認、制御不備、記録不備に関するインシデント記録が `docs/incidents/` に複数存在する。

発生前に停止できたはずのゲート：

応答前制御の指示対象固定ゲート、作業前制御の確認対象列挙ゲート、ツール実行前制御の対象範囲確認ゲート、実行中制御のスコープ逸脱停止ゲート、報告前制御の完了条件確認ゲート、事後監査制御の自己検知ゲート。

再発防止策の作用タイミング：

応答前、作業前、ツール実行前、実行中、報告前、commit 前、commit message 作成時、push 前、インシデント記録作成時。

再発防止策の検証方法：

該当記録が、発生時点、本来作用すべき制御、実際に作用した制御、制御が効かなかった理由、既存再発防止策が効かなかった理由、同種過去インシデント、発生前に停止できたはずのゲート、作用タイミング、検証方法、制御区分、残存リスク、自己検知・自己報告の成否を持つことを pre-commit Hook で確認する。

制御区分：

応答前制御、作業前制御、ツール実行前制御、実行中制御、報告前制御、GitHook制御、事後監査制御。

残存リスク：

チャット上の指示解釈、質問と作業指示の識別、外部環境確認、実行結果確認は、Hook だけでは完全に停止できない。非 Git 制御で作業前および報告前に確認する必要が残る。

自己検知・自己報告の成否：

既存記録本文では、発生前または発生直後に AI/Codex が自己検知して自己報告した事実は記録されていない。事後にインシデント記録として整理されている。
