INC-20260526-007-重大

タイトル：制御系の0からの再設計指示に対して GitHook 偏重の設計を継続した

背景：ユーザーは、AGENTS や Hook やその他制御系を0から再設計すること、原則は常に普遍的かつ恒久的で例外なく変更不可であること、GitHook にすべて詰め込む設計が要件違反であることを指摘していた。

発生インシデントの説明：

制御系の再設計として、`AGENTS.md` に旧来の GitHook 中心設計を引き継ぐ文言を追加した。

しかし、実装と説明の中心は `scripts/agents-compliance-check.ps1` と Git GitHook制御に寄っており、Git 操作を伴わない応答、会話解釈、作業開始前、報告前の制御を、GitHook 以外のアクションタイミングや制御機構として具体化できていなかった。

その後の報告でも、GitHook 前では効果が限定されるという説明に終始し、ユーザーが要求した「常に普遍的かつ恒久的」な制御、および GitHook 以外の Hook / 制御系を含む0からの再設計に答えていなかった。

定量記録：

- 影響を受けたブランチ数：1
- 作成した不適切な commit 数：0
- push 回数：0
- 起動または停止した pipeline 実行数：0
- 変更ファイル数：4
- 復旧 commit 数：0

なぜ発生したのか：

事実確認不足、スコープ変更、設計要件の不保持が原因である。

確認すべきだった事実は、ユーザーが GitHook制御単独では不十分であり、Git 操作を伴わない応答でも違反が多発していると指摘していたこと、0からの再設計対象に AGENTS、Hook、その他制御系が含まれていたこと、原則は GitHook の発火有無に依存しない全制御レイヤで作用しなければならないことだった。

元の指示範囲は、AGENTS、Hook、その他制御系を0から再設計し、原則を常に普遍的かつ恒久的に作用させる制御体系を作ることだった。

不適切に狭めた範囲は、AGENTS への旧来制御文言追加と GitHook の検査強化を中心とする範囲だった。

再発防止として、制御系設計では、GitHook、会話応答前、作業開始前、ツール実行前、ファイル編集前、報告前、事後監査の各アクションタイミングを分離し、どの制御がどのタイミングで作用するかを設計対象として明示する。GitHook制御だけで作用しない領域を、GitHook制御の限界として放置せず、非 Git 制御として設計対象に含める。

何に違反したのか：

第一原則の「明示的に許可がない限り、作業を行わないこと」「不明点や曖昧な点があれば、作業前に必ず確認すること」に対し、再設計の不足範囲を確認せずに対応済みとして報告した点で違反した。

共通解釈規則の「指示対象を、確認なしに別対象へ拡張しないこと」「確認が必要な状態で、確認なしに作業を続行しないこと」「要件を1つでも満たしていない状態で、完了、成功、対応済み、問題なしと扱わないこと」に違反した。

スコープ変更禁止の「ユーザーの明示指示を、一部の語句、単一の違反類型、または作業者に都合のよい範囲へ狭めてはならない」に違反した。

報告制御の「重要な判断、結論、指摘、提案には、それぞれ対応するエビデンスを付すこと」「未確認事項は未確認と明記すること」に対し、GitHook 以外の制御を未設計のまま、制御系再設計として報告した点で違反した。

発生時点：

制御系再設計の設計段階、実装段階、報告前段階で発生した。

本来作用すべき制御：

作業前制御で GitHook 以外の制御系を設計対象へ含め、報告前制御で0からの再設計要件を満たしているか確認すべきだった。

実際に作用した制御：

AGENTS 文言追加と GitHook 検査強化だけが中心になり、非 Git のアクションタイミング別制御は具体化されなかった。

制御が効かなかった理由：

制御対象を Git 操作に現れる成果物へ狭め、会話受信時、応答前、作業開始前、ツール実行前、報告前の実効制御を設計しなかったため。

既存再発防止策が効かなかった理由：

既存策自体が GitHook 偏重であり、その偏重を監査する項目や、GitHook 以外の制御設計を必須化する項目が不足していたため。

同種過去インシデント：

GitHook では止められない会話解釈、作業開始前、報告前の違反が複数回発生していた。

発生前に停止できたはずのゲート：

応答前制御で、ユーザー指示の対象が GitHook制御単独ではなく AGENTS、Hook、その他制御系全体であることを固定していれば停止できた。

作業前制御で、GitHook 以外のアクションタイミングと制御機構を設計対象に列挙していなければ作業開始しない、と判定していれば停止できた。

報告前制御で、Git 操作を伴わない応答、会話解釈、作業開始前、報告前の違反を防ぐ非 Git 制御が具体化されていないことを検出していれば停止できた。

事後監査制御で、直近報告が GitGitHook制御の限界説明に偏り、ユーザーの「GitHook しか考慮していない設計も含めて0から設計しなおせ」という対象に答えていないことを検出していれば停止できた。

再発防止策の作用タイミング：

制御系設計前、実装前、報告前、事後監査制御時に作用する。

再発防止策の検証方法：

制御系変更時に、会話受信時、応答生成前、作業開始前、ツール実行前、ファイル編集前、報告前、GitHook、事後監査の各タイミングへ制御が割り当てられていることを確認する。

制御区分：

非 Git 制御は応答前制御、作業前制御、ツール実行前制御、ファイル編集前制御、報告前制御、事後監査制御。GitHook制御は pre-commit、commit-msg、pre-push。

残存リスク：

Codex の応答生成前制御はリポジトリ内 Hook では機械停止できないため、AGENTS の制御系0設計と応答前自己監査に依存する領域が残る。

自己検知・自己報告の成否：

自己検知できず、ユーザー指摘後に発覚した。

制御不備または原則不遵守：

制御不備および原則不遵守である。

会話応答前、作業開始前、ツール実行前、報告前の違反は、応答前制御、作業前制御、ツール実行前制御、ファイル編集前制御、報告前制御で停止すべきである。

今回の不備は、GitHook による停止条件の不足だけでなく、GitHook 以外の Hook / 制御系を設計対象から実質的に外した設計不備である。

対応策としての制御系修正：

GitHook制御だけへの追加修正は、このインシデントの再発防止として不十分である。制御系設計では、GitHook制御は Git 状態、staged diff、commit message、pre-push stdin を対象にする決定的検査として限定し、非 Git 応答・作業開始前・ツール実行前・報告前の制御を別アクションタイミングとして設計する。

対応策としての関連ドキュメント修正：

この記録により、GitHook 偏重の再設計が要件違反であること、制御系再設計には GitHook 以外のアクションタイミングと制御機構を含める必要があることを明記した。
