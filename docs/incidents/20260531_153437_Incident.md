INC-20260531-007-重大

タイトル：再発防止策の実施指示に対して制御系修正を除外した

背景：
ユーザーは「すべてのインシデント記録を行い再発防止策を実施しろ」と指示した。さらに、直後に「なぜ、再発防止策を実施しろと指示したのに、制御系を含めない？」と指摘した。

発生インシデントの説明：
再発防止策の実施指示に対し、制御系ファイルの明示指示がないと解釈し、`AGENTS.md` と `scripts/agents-compliance-check.ps1` の修正を除外した。

定量値：影響を受けたブランチ数は 1。作成した不適切な commit 数は 0。push 回数は 0。起動または停止した pipeline 実行数は 0。制御系修正を除外した回答回数は 1。復旧 commit 数は 0。

なぜ発生したのか：
「再発防止策を実施しろ」を、関連ドキュメント更新だけへ不当に狭めたため。

何に違反したのか：
第四原則の「指示に従うこと」「指示を曲解しないこと」に違反した。スコープ変更禁止の「ユーザーの明示指示を、一部の語句、単一の違反類型、または作業者に都合のよい範囲へ狭めてはならない」に違反した。

元の指示範囲：
すべてのインシデント記録を行い、再発防止策を実施すること。

不適切に狭めた範囲：
インシデント記録と関連ドキュメント修正だけに限定し、制御系修正を除外した。

再発防止策：
`AGENTS.md` に、AGENTS を人間ユーザーへの拘束として扱わない制御、正規手順依頼時に正規手順外の停止を追加しない制御、インシデント後の事実確認で過去作業指示を引き継がない制御、複数質問へ全件回答する制御を追加した。`scripts/agents-compliance-check.ps1` に同制御の必須マーカーを追加した。

発生時点：
5件のインシデント記録を作成した後、対応策としての制御系修正を不要と報告した時点。

本来作用すべき制御：
応答前制御で、再発防止策の実施指示が制御系修正を含み得ることを固定する必要があった。作業前制御で、制御系ファイル変更の明示指示が今回の再発防止策実施指示に含まれることを扱う必要があった。

実際に作用した制御：
関連ドキュメント修正だけを実施し、制御系修正を除外した。

制御が効かなかった理由：
制御系ファイル保護を、ユーザーの再発防止策実施指示を縮小する根拠として扱ったため。

既存再発防止策が効かなかった理由：
既存策は制御系ファイルの無断変更防止に偏り、ユーザーが再発防止策実施を指示した場合に制御系修正を含める判断として作用しなかったため。

同種過去インシデント：
`docs/incidents/20260526_153939_Incident.md` の制御系設計指示を不当に狭めたインシデント、`docs/incidents/20260528_132316_Incident.md` のスコープ不当限定と同種である。

発生前に停止できたはずのゲート：
応答前制御、作業前制御、ファイル編集前制御、報告前制御で停止できた。

再発防止策の作用タイミング：
ユーザーが再発防止策、制御、AGENTS、Hook、正規手順、指揮系統に関する対応を指示した時点で作用する。

再発防止策の検証方法：
再発防止策の実施指示に対し、制御系修正要否を明示し、必要な場合は `AGENTS.md` と hook の差分が存在することを確認する。

制御区分：
応答前制御、作業前制御、ファイル編集前制御、報告前制御、GitHook制御。

残存リスク：
制御系ファイル保護を、制御系修正が必要な明示指示まで拒む理由として誤用するリスクが残る。

自己検知・自己報告の成否：
自己検知は失敗した。ユーザー指摘後に認めた。

対応策としての制御系修正：
`AGENTS.md` と `scripts/agents-compliance-check.ps1` を修正し、今回の各違反に対する制御を追加した。

対応策としての関連ドキュメント修正：
`docs/development-records/reporting-and-authorization-gate.md` を更新し、再発防止策の実施時に制御系を不当に除外しない前提となる報告前・作業前ゲートを補強した。

復旧案：
前回作成した5件のインシデント記録の制御系修正欄を、制御系修正実施済みの内容へ更新する。
