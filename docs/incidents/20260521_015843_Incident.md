# 人間の手動 push を拒否する pre-push 設計

## 発生日時

2026-05-21 01:58:43

## 区分

違反

## 概要

AI/Codex の protected branch push を制御する目的で実装した pre-push GitHook制御が、人間の手動 push も拒否した。

この制御は、AI への作業制御を Git GitHook制御に直接実装したことで、実行者を区別せず全 push に適用された。

## 影響範囲

- `.githooks/pre-push`
- `scripts/agents-compliance-check.ps1`
- `AGENTS.md`

## 確認した事実

- push 失敗画面に `AGENTS HOOK STOP: push to protected branch is forbidden without AGENTS_ALLOW_PROTECTED_PUSH=1` と表示された。
- `.githooks/pre-push` は `scripts/agents-compliance-check.ps1 -Mode pre-push` を実行する。
- `scripts/agents-compliance-check.ps1` の pre-push 制御は、実行者が人間か AI かを区別せず、protected branch push を停止していた。
- `AGENTS.md` は `dev` または `main` への push を `AGENTS_ALLOW_PROTECTED_PUSH=1` がない場合に停止すると記載していた。

## 原因

AI/Codex への制御を、全 Git 実行者に適用される hook 側の強制停止として設計した。  
そのため、人間の手動操作を制御対象外にする要件を満たしていなかった。

## 即時対応

- pre-push hook の protected branch 停止を AI/Codex 用の opt-in 制御に変更する。
- `AGENTS_AI_PROTECTED_PUSH_GUARD=1` が設定されている場合のみ、protected branch push を強制停止対象にする。
- 人間の手動 push では GitHook制御が warning を出すだけで停止しないようにする。
- `AGENTS.md` の pre-push 手順を、人間の手動 push を停止しない設計に改める。

## 再発防止

- AI への制御を Git GitHook制御に実装する場合、人間の手動操作へ同じ停止条件が適用されないかを設計時に確認する。
- 人間と AI を GitHook制御のみで確実に区別できない場合、人間の操作を止める設計にしない。
- AI/Codex が protected branch push を行う場合は、AI 側が `AGENTS_AI_PROTECTED_PUSH_GUARD=1` を設定して opt-in ガードを使う。

## 対応策としての制御系修正：

`scripts/agents-compliance-check.ps1` の `Assert-ProtectedPushAllowed` を変更し、`AGENTS_AI_PROTECTED_PUSH_GUARD=1` が設定されていない場合は protected branch push を停止せず warning のみ出力する。

`AGENTS_AI_PROTECTED_PUSH_GUARD=1` が設定されている場合は、従来通り `AGENTS_ALLOW_PROTECTED_PUSH=1` がない protected branch push を停止する。

## 対応策としての関連ドキュメント修正：

`AGENTS.md` の `pre-push` 手順を変更し、人間の手動 push を GitHook制御で停止してはならないことを明記する。

AI/Codex が protected branch push を行う場合のみ、`AGENTS_AI_PROTECTED_PUSH_GUARD=1` と `AGENTS_ALLOW_PROTECTED_PUSH=1` を使用する手順に変更する。

## 未確認事項

- 人間の手動 push で warning のみ出力され、push が停止しないこと。
- AI/Codex が `AGENTS_AI_PROTECTED_PUSH_GUARD=1` を設定した状態で、`AGENTS_ALLOW_PROTECTED_PUSH=1` なしの protected branch push が停止すること。
