# 原則厳守 GitHook制御が指示違反を停止できていなかったインシデント

## 概要

ユーザーは以前、原則を厳守させるための hook 設計を指示していた。

しかし現状の hook は、外部ツール追加の無許可実施、推測に基づく作業判断、確認不足による報告矛盾、侵害復旧をローカル修正で停止する行為を十分に停止できていなかった。

## 発生した事実

- `scripts/agents-compliance-check.ps1` は AGENTS.md の必須マーカー、成果物ドキュメントの evidence label、インシデント記録形式、staging source revision 記録などを検査していた。
- しかし、GNU gettext の `apt-get install -y gettext` 追加 commit `43d3fe7` は GitHook制御を通過した。
- 推測に基づいて CodeBuild でも GNU gettext が必要だと判断した作業は GitHook制御で停止されなかった。
- 侵害復旧をローカル修正で停止した行為は GitHook制御で停止されなかった。
- 事実確認不足を含む報告矛盾は GitHook制御で予防できなかった。
- 影響を受けたブランチは `dev`、`origin/dev`、作業ブランチ `v2.0.20`、`v2.0.21`、`v2.0.22`、`v2.0.23`、`v2.0.24`、`v2.0.25`。

## 侵害内容

- 原則厳守を目的とした GitHook制御が、原則違反を十分に停止できなかった。
- 外部ツール追加の明示許可確認を機械的に要求していなかった。
- 推測、確認不足、侵害復旧停止という違反類型を hook の検査対象にしていなかった。
- hook の存在により原則が守られているように見えるが、実際には重要な違反を通過させた。

## 影響

- 無許可外部ツール追加が `origin/dev` に反映された。
- staging pipeline が未承認外部ツール追加を含む revision を実行した。
- ユーザーが再指摘するまで、hook 設計の不足が顕在化しなかった。

## 原因

- GitHook制御が形式検査に偏り、作業判断の前提事実確認や外部資産取得コマンド追加を検査していなかった。
- 原則不遵守を検出する hook 要件が、具体的な禁止パターンや承認マーカーに落とし込まれていなかった。
- 制御不備をインシデント化した場合に、該当制御レイヤで停止できる再発防止策を GitHook制御に追加する制御がなかった。
- 事実確認不足。
- 確認すべきだった事実は、GitHook制御が外部資産取得コマンド追加、推測を含むインシデント記録、制御不備を含むインシデント記録を停止できるかである。
- 再発防止として、該当するインシデント記録に 制御系修正または該当制御レイヤで停止できない理由を要求する検査を追加した。

## 対応策としての制御系修正：

`scripts/agents-compliance-check.ps1` に、外部資産取得コマンド追加を `AGENTS_ALLOW_EXTERNAL_ASSET_CHANGE=1` なしで停止する検査を追加済み。

さらに本対応で、制御不備または原則不遵守をインシデント記録する場合、`scripts/agents-compliance-check.ps1` の修正または該当制御レイヤで停止できない理由を要求する検査を追加した。

## 対応策としての関連ドキュメント修正：

`AGENTS.md` の pre-commit 制御に、制御不備または原則不遵守を記録する場合、該当制御レイヤで停止できる再発防止策を `scripts/agents-compliance-check.ps1` に追加しない限り停止することを追記した。

該当制御レイヤで停止できないな場合は、その理由を同じインシデント記録に明記することも追記した。

## 復旧対応

GNU gettext 導入行は復旧 commit `1e81ab51e9e0455e6a8fe6c976148d09ce5f3b78` で削除済み。

復旧 merge commit `9d6f3dca8622de5bff0451c2bb897c9c97995617` は `origin/dev` に push 済みで、staging pipeline execution `35c428d5-4f6d-467c-a059-b3e2a447f932` は source revision `9d6f3dca8622de5bff0451c2bb897c9c97995617` で `Succeeded`。

## 現在の状態

- 制御不備に対する追加検査を `scripts/agents-compliance-check.ps1` に実装した。
- 本インシデント記録は作業ブランチ `v2.0.25` 上で作成中。

発生時点：

2026-05-28 の復旧作業時点で、既存記録本文に基づき追補した。発生時点は、既存記録の「発生インシデントの説明：」および「なぜ発生したのか：」に記載された手順逸脱、完了誤認、確認漏れ、または無許可操作の発生時点である。

既存再発防止策が効かなかった理由：
