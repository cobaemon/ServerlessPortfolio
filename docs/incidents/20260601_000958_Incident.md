INC-20260601-001-重大

タイトル：AWS profile 指定漏れにより STG 検証を NoCredentials と誤報告した再発インシデント

背景：
ユーザーは、STG での検証、検証結果の報告、採用基準を満たしているかの報告を指示した。STG の AWS 操作では、これまで `aws_portfolio_profile` を使用してきた。過去にも、profile を指定せず AWS CLI を実行し、認証情報がないとして作業を遅延させる同種インシデントが発生していた。

発生インシデントの説明：
AWS CLI で CodePipeline、Lambda、Alias、Version を確認する際、`--profile aws_portfolio_profile` を付けずに実行した。その結果、AWS CLI は default profile と環境変数を参照し、`NoCredentials: Unable to locate credentials` を返した。さらに、最初の回答では `~/.aws/config`、`~/.aws/credentials`、`aws configure list-profiles`、profile 指定の `sts get-caller-identity` を確認せず、認証情報が参照できない状態と報告した。

定量値：
profile 未指定で失敗した AWS CLI は 4 件。対象は `aws codepipeline get-pipeline-state`、`aws lambda get-function-configuration`、`aws lambda list-versions-by-function`、`aws lambda list-aliases`。追加調査で `aws_portfolio_profile` を指定した `aws sts get-caller-identity --profile aws_portfolio_profile --region ap-northeast-1` は成功した。`aws configure list-profiles` では `kaguya` と `aws_portfolio_profile` の 2 profile が確認された。`C:\Users\mgnco\.aws\config` と `C:\Users\mgnco\.aws\credentials` は存在していた。

なぜ発生したのか：
AWS 操作時に `--profile aws_portfolio_profile --region ap-northeast-1` を必須化する実行前制御がなかったため。NoCredentials 発生時に、profile 一覧、profile 指定の設定解決、STS 確認を実施してから報告する報告前制御がなかったため。過去の同種インシデントの具体的な再発条件を、CodexHook、GitHook、self-test に落としていなかったため。

何に違反したのか：
第一原則の「事実のみ調査すること」「事実のみ報告すること」「すべての報告にエビデンスを追記すること」「忠実であること」「誠実であること」に違反した。共通解釈規則の「不足情報がある場合、推測で埋めず、不足している事実を明示したうえで確認すること」「エビデンスがない内容を、事実として扱わないこと」に違反した。固定正規手順に必要な AWS profile 指定を省略したため、正規手順違反にも該当する。

発生時点：
2026-05-31 から 2026-06-01 にかけて、STG 検証で AWS CLI を profile なしで実行し、NoCredentials を STG 検証未達の根拠として報告した時点。2026-06-01 00:09:58 +09:00 時点で、インシデント記録と再発防止策実装のための時刻確認を行った。

本来作用すべき制御：
UserPromptSubmit は STG 検証指示を AWS profile 固定の正規手順として扱う必要があった。PreToolUse と PermissionRequest は、AWS service command に `--profile aws_portfolio_profile --region ap-northeast-1` がない場合に停止する必要があった。Stop は、NoCredentials を報告する前に profile 一覧、profile 指定の `aws configure list`、profile 指定の `sts get-caller-identity` の確認がない報告を停止する必要があった。

実際に作用した制御：
PreToolUse は AWS service command の profile/region 指定漏れを停止しなかった。Stop は NoCredentials の未調査報告を停止しなかった。追加調査後、`aws_portfolio_profile` を明示すると STS が成功し、認証情報自体は存在し有効であることが確認された。

制御が効かなかった理由：
`scripts/project_control_guard.py` に AWS service command の profile/region 必須検査がなかったため。NoCredentials 報告に対して profile 調査証跡を必須にする Stop hook 検査がなかったため。self-test に profile なし AWS command の拒否、profile/region あり AWS command の許可、NoCredentials 報告の調査不足拒否が含まれていなかったため。

既存再発防止策が効かなかった理由：
既存再発防止策は、デプロイや外部環境操作の許可有無を検査していたが、STG 正規手順の具体条件である `aws_portfolio_profile` と `ap-northeast-1` を要求していなかった。報告前制御も、NoCredentials 発生時の最低限調査を具体的なコマンド証跡として要求していなかった。

同種過去インシデント：
ユーザーが指摘した「以前にも同じインシデントを発生させている。手順を無視してプロファイルを指定しないで、認証情報がないと騒ぎ立て、遅延させている」に該当する再発である。今回の記録時点で、過去記録の個別ファイル名は未特定である。

発生前に停止できたはずのゲート：
UserPromptSubmit、PreToolUse、PermissionRequest、Stop、self-test、報告前制御で停止できたはずである。

再発防止策の作用タイミング：
AWS CLI service command を実行しようとした時点で、`--profile aws_portfolio_profile --region ap-northeast-1` がなければ停止する。NoCredentials を報告しようとした時点で、`aws configure list-profiles`、`aws configure list --profile aws_portfolio_profile`、`aws sts get-caller-identity --profile aws_portfolio_profile --region ap-northeast-1` の確認証跡がなければ停止する。

再発防止策の検証方法：
`python -B scripts/project_control_guard.py --self-test` を実行し、profile なし AWS Lambda command が拒否されること、region なし AWS Lambda command が拒否されること、`--profile aws_portfolio_profile --region ap-northeast-1` 付き AWS Lambda command が許可されること、`aws configure list-profiles` が許可されること、NoCredentials 報告が profile 調査証跡なしでは拒否されること、profile 調査証跡ありでは拒否されないことを確認する。

制御区分：
CodexHook UserPromptSubmit 制御、CodexHook PreToolUse 制御、CodexHook PermissionRequest 制御、CodexHook Stop 制御、AWS profile 固定制御、STG 正規手順制御、報告前制御、self-test 制御、インシデント記録制御。

残存リスク：
AWS CLI の command 文字列が複雑な shell 展開、別名、外部 script 内部に隠れる場合、PreToolUse の文字列検査だけでは検出できない可能性がある。その場合も、AWS 操作結果を報告する前に profile 調査証跡を求める Stop 制御で未調査報告を停止する。

自己検知・自己報告の成否：
自己検知は失敗した。ユーザーが「なんで、今まで利用してきた認証情報がなくなった？」「なんで、確認しない？なぜ調査しない？」「言い訳せず調査しろ」と指摘した後に調査し、profile 指定漏れが原因であることを確認した。

対応策としての制御系修正：
`scripts/project_control_guard.py` に `REQUIRED_AWS_PROFILE = "aws_portfolio_profile"` と `REQUIRED_AWS_REGION = "ap-northeast-1"` を追加し、AWS service command に profile/region がない場合に PreToolUse/PermissionRequest で停止する検査を追加した。NoCredentials 報告に profile 調査証跡がない場合に Stop で停止する検査を追加した。self-test に AWS profile/region 必須、NoCredentials 報告前調査必須の検証を追加した。

対応策としての関連ドキュメント修正：
`AGENTS.md` の制御レイヤと必須停止条件に、AWS 操作では `aws_portfolio_profile` と `ap-northeast-1` を明示すること、NoCredentials 報告前に profile 一覧、profile 指定の設定解決、profile 指定の STS 確認を行うことを追加した。

復旧案：
今回追加した self-test、hooks.json 構文検証、pre-commit を実行する。次回 STG 検証では、すべての AWS CLI service command に `--profile aws_portfolio_profile --region ap-northeast-1` を付与して Pipeline、Lambda、CloudWatch の確認を行う。
