# 開発記録

このディレクトリは、開発過程で発生または使用する調査、方針、対応案、検討記録を配置する。

## 配置ルール

開発過程で発生または使用する調査報告、方針策定、対応案、比較検討、実装前の検討記録を配置する。

各開発記録には、確認対象、確認結果、未確認事項を記載する。

確認対象には、作業前または報告前に確認すべきローカルファイル、AWS などの外部環境、公式ドキュメント、実行ログ、既存仕様を記載する。

確認結果には、実際に確認した事実と確認元を記載する。確認可能だが未確認の事項は、確認結果ではなく未確認事項に記載する。

プロジェクト仕様、設計、設定、運用、デプロイ、開発環境など、継続的に参照するプロジェクトドキュメントは `docs/` 直下に配置する。

AI 作業進捗、作業途中の AI 用 TODO、AI が使用する確認記録は `docs/ai-progress/` に配置する。

作業中に発生した指示違反、手順逸脱、影響発生、復旧対応の記録は `docs/incidents/` に配置する。

このディレクトリの記録を恒久的な仕様または運用手順として採用する場合は、採用後の内容を `docs/` 直下の該当プロジェクトドキュメントへ反映する。

仕様、運用、手順、デプロイ、pipeline、外部環境に関する開発記録を正式採用または本番反映した場合は、同じ記録内に `ドキュメント反映状況` を設け、反映済み、未反映、対象外の区分と対象ドキュメントを記載する。
